# Jnf

Веб-генератор XML-крафтів для Juno: New Origins у GitHub Codespaces.

Застосунок реалізує ширший сценарій, ніж лише ракети-перехоплювачі: він підтримує профілі для ПРО, ракет-носіїв, дослідницьких ракет і орбітальних апаратів.

## Що вже є

- Flask-сервер з endpoint'ом для генерації XML.
- Шаблонний Python-генератор на базі валідної структури craft XML для Juno, без вигаданих тегів.
- Веб-форма, де користувач вводить лише назву крафту і текстовий опис бажаного результату.
- Інструкції для перенесення XML у гру на Android через MT Manager.

## Швидкий старт

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python app.py
```

Після запуску відкрийте http://127.0.0.1:8000.

## AI-розбір запитів

Застосунок тепер підтримує структурований розбір опису крафта через OpenAI Responses API.

- Якщо задано OPENAI_API_KEY, сервер спробує витягнути variant_name, profile_id, stages, theme і множники тяги/палива з вільного тексту.
- Якщо ключа немає або API недоступний, усе автоматично падає назад на локальні евристики без поломки генерації.
- Типова конфігурація: OPENAI_MODEL=gpt-5.4.
- У веб-інтерфейсі є AI-аналіз, який показує, що саме було витягнуто із запиту ще до завантаження XML.

### Режим великих шаблонів (20k+ рядків)

Якщо потрібен справді деталізований крафт (умовно 20k+ рядків XML), додайте готовий шаблон у бібліотеку:

- `templates/craft_library/thaad.xml`
- `templates/craft_library/patriot_pac3.xml`
- `templates/craft_library/iskander_m.xml`

Або задайте свою директорію через `CRAFT_TEMPLATE_DIR`.

Коли в запиті є фрази на кшталт "копія", "replica", "дуже чіткий", генератор пріоритезує шаблон. Якщо файл шаблону знайдено, повертається саме він (із підстановкою `Craft name`). Якщо не знайдено, використовується процедурний fallback.

Приклад:

```bash
export OPENAI_API_KEY=your_key_here
export OPENAI_MODEL=gpt-5.4
python app.py
```
